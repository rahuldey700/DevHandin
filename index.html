Overview

This project showcases a car shopping list page where users can select cars they like and add them to their favorites. They can filter cars by type, color, and price. Inspiration was drawn from three websites: Rotten Tomatoes, Yelp, and Goodreads.

Rotten Tomatoes Reset Filters feature

With multiple layers of filters, it gets difficult to get back to the original and often useful default list. A Reset filters helps the user get back to the initial view once they have done playing around with other sorting and filtering mechanisms. After sorting different movies in my Rotten Tomatoes list, such as filtering for 1-star-rated Action Movies, I might want to get back to my original list of movies. The Reset filter is very helpful for this.

GoodReads Batch Edit Filters

Especially when there is a large number of items, the user might want to edit a large number of items at once. For example, I might want to delete 4 books because I read them or watched the movies. The Batch Edit Filter helps with this functionality.

Yelp’s Filter by Category

For a varied list of items, the user might want to categorize them based on a common factor. For example, in my Yelp list, I have restaurants, barbers, cafes, and shops. I might only be looking for restaurants to have dinner in. So I can filter by restaurants and even further filter by ratings and price.

My Interface:

I am going to build a list of cars. The user will have the ability to choose different cars and then sort them by several factors such as price, size/type, color, etc.

  1. Filter by key categories that are pertinent to the product. I’ve chosen Car Type, Color, and Price.
  2. A reset filter option so the user can go back to their previous options
  3. Ability to add/save products to a list
  4. An informative view for the user so they can see the details at a glance and make a decision. I’ve chosen a card view that will contain images of the car along with some key information such as price and specs description.
